# 实车录音声学统计脚本说明

## 用途

分析实车录音的两个关键声学参数：**混响时间 T60** 和 **信噪比 SNR**，
用于指导 TTS 离线增强 pipeline 的 RIR 选取和加噪强度配置。

---

## 参数含义

### SNR（信噪比）

| 范围 | 含义 |
|------|------|
| > 20 dB | 很干净，接近录音室 |
| 10~20 dB | 轻度噪声，正常车载安静场景 |
| 5~10 dB | 中度噪声，行驶中或空调开启 |
| < 5 dB | 重噪声，高速行驶 / 风噪 |

**如何用于指导增强**：若实车录音 p50 SNR ≈ 10dB，则 TTS 加噪时应优先覆盖 5~15dB 范围，而非当前单一档位。

### T60（混响时间）

| 范围 | 含义 |
|------|------|
| < 100 ms | 几乎无混响，强吸音车舱 |
| 100~300 ms | 典型车舱范围 |
| > 300 ms | 混响较重，SUV / 大型车厢 |

**如何用于指导增强**：若实车录音 p50 T60 ≈ 200ms，则从 `reverb_pak_real_135m.npy` 中优先选 T60 在 150~250ms 范围的 RIR 子集，而非随机使用全库。

## 算法原理

### SNR 估计：能量 VAD 法

无需干净参考信号，步骤：

1. 将音频切成 20ms 帧
2. 计算每帧 RMS 能量
3. 按 RMS 升序排列，最低 20% 的帧认为是**噪声帧**，最高 20% 认为是**语音帧**
4. SNR = 20 × log₁₀(语音帧平均 RMS / 噪声帧平均 RMS)

**局限**：若录音几乎没有静音段，噪声帧估计偏高，SNR 会被低估。
适用前提：每条录音中有一定比例的非语音段（起止静音）。

### T60 估计：能量衰减斜率法

原理来自 Schroeder 积分的简化版，步骤：

1. 计算短时 RMS 包络（10ms hop）
2. 找到能量峰值点（语音结束处附近）
3. 对峰值之后的衰减段做线性回归（dB vs. 时间）
4. 由斜率推算衰减 60dB 所需时间 = T60

**局限**：
- 对持续语音（中间无明显停顿）估计不准
- 精度约 ±50ms，够用于 RIR 子集筛选，不适合精确声学测量
- 结果在 50ms~2s 范围外会被过滤掉（返回 None）

## 如何使用统计结果

拿到 p25/p50/p75 后，向 pipeline 维护者（ssyan2）提需求时给出具体参数：

```
实车录音声学特性：
  SNR: p25=9dB / p50=11dB / p75=14dB  → 建议加噪范围 5~15dB
  T60: p25=0.17s / p50=0.20s / p75=0.22s → 建议选 T60 150~250ms 的 RIR 子集

当前 TTS 增强配置不覆盖上述范围，是实车准召低的根因之一。
```

---

## 注意事项

- 脚本对多通道音频自动取均值转单通道
- 非 16kHz 采样率会自动 resample（依赖 librosa，速度较慢）
- 若大量文件返回 T60=None，说明这批录音衰减不明显（可能在安静室内录制或混响极短）
